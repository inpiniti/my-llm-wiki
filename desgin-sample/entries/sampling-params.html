№ 02 중급 해결됨

Temperature·Top-k·Top-p, 결국 무엇을 만지는가

세 노브는 모두 출력의 “다양성”에 영향을 주지만, 확률 분포의 다른 부분을 깎는다. 직접 슬라이더로 본 뒤에야 차이가 분명해졌다.

관련 프로젝트SQL 생성 에이전트
증상간헐적 환각
시도3가지 파라미터 매트릭스
정착T=0.3 · top_p=0.9

증상

SQL 생성 에이전트가 99% 잘 작동하다가 가끔 존재하지 않는 컬럼명을 만들어냈다. 같은 질문을 두 번 던지면 한 번은 맞고 한 번은 틀렸다. “창의성”이 필요 없는 작업이었으므로 sampling을 조여야 했다.

세 개의 노브, 한 분포

디코딩 단계에서 모델은 다음 토큰에 대한 확률 분포를 가지고 있다. 세 파라미터는 이 분포를 다음과 같이 변형한다:

  • Temperature (T): 분포 자체를 평탄/날카롭게 만든다. softmax(logit / T). T가 낮으면 가장 확실한 토큰에 더 집중.
  • Top-k: 확률 상위 k개만 남기고 나머지 컷. k=1이면 greedy와 같다.
  • Top-p (nucleus): 확률 누적합이 p에 도달할 때까지의 토큰만 남긴다. 분포가 뾰족하면 적게, 평평하면 많이 살아남는다.

직접 만져보기

가상의 문장 “오늘 날씨가 ___” 다음에 올 후보 10개의 logit을 미리 정해 두었다. 세 슬라이더를 움직이면 각 단계의 효과를 분리해서 볼 수 있다.

샘플링 시각화 interactive
오늘 날씨가
Temperature 1.00
Top-k 0
Top-p 1.00
슬라이더를 움직이고 한 번 뽑아 보세요.

슬라이더에서 본 직관

규칙 1

Temperature는 형태를 바꾼다. 낮추면 분포가 한 쪽으로 쏠리고, 높이면 평평해진다. 분포의 모양 변화이지, 잘라내는 것이 아니다.

규칙 2

Top-k는 칸을 센다. 분포가 평평하든 뾰족하든 정확히 k개만 남긴다. 컨텍스트에 따라 “적절한 k”가 다르다는 게 약점.

규칙 3

Top-p는 적응적이다. 모델이 확신 있을 때(분포가 뾰족) 후보가 적게, 모호할 때 후보가 많이 살아남는다. 그래서 보통 top-k보다 top-p가 권장된다.

내 프로젝트에서의 정착값

여러 매트릭스를 돌려본 후, 작업의 성격에 따라 두 세트로 정리했다:

// 사실 추출, SQL/JSON 생성, 분류
T = 0.0  ~  0.3
top_p = 0.9
top_k = 0  (사용 안함)

// 요약, 카피라이팅, 대화
T = 0.7  ~  1.0
top_p = 0.95
top_k = 0
현장 메모. T=0이 결정론적인 것은 아니다. 동률 발생, 부동소수점, 그리고 KV cache 차이로 같은 입력도 결과가 흔들릴 수 있다. 진짜 reproducibility가 필요하면 seed 고정 + greedy + cache 끄기까지 가야 한다.

남은 의문

Min-p, typical sampling, mirostat 같은 “분포 모양 자체”를 다루는 새로운 sampler들은 실전에서 top-p보다 더 안정적인가? 작은 모델일수록 효과가 더 크다는 보고가 있다.
