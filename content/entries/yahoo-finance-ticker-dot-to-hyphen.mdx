## 관찰 / 문제

포트폴리오 화면에서 `BRK.B` 종목의 가격이 표시되지 않았다.
백엔드 캐시에 `close`가 없어서 Yahoo Finance로 보충 요청하는 로직이 있었는데,
`BRK.B`를 그대로 URL에 사용했더니 `null`을 반환했다.

## 왜 그런가 / 원인

Yahoo Finance의 실제 티커 식별자는 점(`.`) 대신 하이픈(`-`)을 사용한다.

- Yahoo Finance 내부: `BRK-B`
- KIS/한투에서 내려오는 데이터: `BRK.B` (또는 `BRK/B`)

따라서 `encodeURIComponent('BRK.B')`를 해도 점은 인코딩되지 않아서
`/v8/finance/chart/BRK.B`로 요청 → 404 또는 빈 응답.

## 해결

```js
// lib/portfolioApi.js - fetchClosePrices
const yfTicker = ticker.replace(/\./g, '-');
const res = await fetch(
  `${YF_BASE}/v8/finance/chart/${encodeURIComponent(yfTicker)}?interval=1d&range=1d`,
  { headers: { 'User-Agent': 'Mozilla/5.0' } },
);
```

<Callout label="결론">
Yahoo Finance API에 티커를 넘길 때는 `.`을 `-`로 변환해야 한다.
클래스 주식(BRK.B, BF.B 등) 또는 특수 상품에서 자주 발생하는 패턴이므로 보편 규칙으로 적용.
</Callout>

## 남은 의문

- 슬래시(`/`)가 포함된 티커(KIS 응답에서 BRK/B 형태)는 별도 처리 필요한지? → normalizeTicker에서 이미 `/` 제거하므로 WS 매칭은 문제 없음. 가격 조회에서는 `/`가 오는 경우 확인 필요.
