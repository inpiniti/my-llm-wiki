## 관찰 / 문제

철거 계량기 4종(GTG-G6R, G-10RG, G-650RF, G-16RC)의 바코드로 설치계량기 기물번호를 역산해야 했다. 바코드 구조가 문서화돼 있지 않아 `init.json` 및 기존 코드를 분석해야 했다.

## 바코드 위치별 의미

```
바코드 예시: 219511100395
위치:        0123456789...

pos 0-1  : 연도 (2자리)          예) "21" → 2021년
pos 2    : 형식 코드              "1" = 막식, "2" = 터빈
pos 3    : 제조사 키 (1자리)      숫자 또는 알파벳
pos 4-5  : 등급 코드 (2자리 int) 숫자열로 인코딩된 등급 인덱스
pos 6    : 타입 코드              "1" = 우(20), "2" = 좌(10)
pos 7+   : 일련번호
```

### 등급 코드 → 등급 매핑

| 코드 | 등급 |
|------|------|
| 01   | 10   |
| 02   | 16   |
| 03   | 25   |
| 04   | 40   |
| 05   | 65   |
| 06   | 100  |
| 07   | 160  |
| 08   | 250  |
| 09   | 400  |
| 10   | 650  |
| 11   | 1000 |

### 제조사 키 → makerNum 매핑

| 키 | makerNum |
|----|----------|
| 0  | 10025    |
| 1  | 10039    |
| 5  | 10044    |
| 9  | 10008    |
| A  | 10042    |

*(다른 키는 `init.json`의 maker 배열에서 확인)*

### 타입 코드 설명

- `1` = 우측 연결, 유량 20
- `2` = 좌측 연결, 유량 10

## 기물번호 산출

```
mtrIdNum = barcode.substring(0, 2) + barcode.substring(7)
```

즉, **연도 2자리 + 일련번호부분** 이다.

### 실제 사례

| 철거 바코드    | 모델    | 등급 | 형식 | 타입 | 기물번호       |
|---------------|---------|------|------|------|---------------|
| 219511100395  | GTG-G6R | 40   | 막식 | 우   | 21100395      |
| 175908117     | G-10RG  | 16   | 막식 | 우   | 17117         |
| 18T189000001  | G-650RF | 1000 | 터빈 | 우   | 18000001      |
| 1801834282    | G-16RC  | 25   | 막식 | 우   | 1834282       |

> `175908117`: pos3='5'(makerNum 10044), pos4-5='08'(등급250? 또는 모델별 별도 해석 필요)  
> 짧은 바코드는 일부 위치가 생략될 수 있음 — 모델 시리즈별로 구조가 다를 수 있다.

## 해결

바코드를 받으면 위 표를 참조해 위치별로 파싱한 뒤, `mtrIdNum = year + serial` 공식으로 기물번호를 산출한다. `init.json`의 maker/grade 배열이 공식 매핑 소스다.

<Callout label="결론">
대용량 계량기 바코드는 위치 기반 인코딩이다. 기물번호는 앞 2자리(연도) + 7번째 이후(일련번호) 조합. 등급·제조사 코드는 `init.json` 배열에서 역인덱스로 찾는다. 바코드 길이가 짧은 구형 모델은 위치 오프셋이 달라질 수 있다.
</Callout>

## 남은 의문

- 바코드가 짧은 경우(9자리 미만) 위치 오프셋이 그대로 적용되는지 불명확
- 터빈식(pos2='2')과 막식(pos2='1') 이외 형식 코드 존재 여부
- `init.json` 이외의 공식 바코드 규격 문서 존재 여부
