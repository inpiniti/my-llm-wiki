## 관찰 / 문제

커밋 b2e00e51에서 `DEFAULT_PAYLOAD`에 `mtrDetaLoc`(상세위치)를 추가했다. 그러나 실제 FMS3003 요청 로그를 확인하니 `mtrDetaLoc`가 전혀 포함되지 않았다.

```
// 로그에서 mtrDetaLoc 없음
FMS3003 request: { mtrNum: ..., indicationAmt: ..., indicationAmtYn: ..., ... }
```

## 왜 그런가 / 원인

`DEFAULT_PAYLOAD`는 **페이지 초기화 시 `page.payload`를 채우는 기본값 객체**다. UI가 이 값을 읽어 화면에 표시하는 데 쓴다.

반면 FMS TR 요청 바디는 `sendChangeResult`(또는 유사 함수) 안에서 **명시적으로 나열된 필드만** 포함된다. `page.payload` 전체를 그대로 전송하는 구조가 아니다.

```javascript
// DEFAULT_PAYLOAD 에 추가했을 때 — 화면엔 뜨지만 서버엔 안 감
const DEFAULT_PAYLOAD = {
  mtrNum: '',
  indicationAmt: '',
  indicationAmtYn: 'N',
  mtrDetaLoc: '',   // ← 여기만 추가해서는 부족
};

// sendChangeResult 내부 — 여기에도 반드시 포함해야 함
function sendChangeResult(data) {
  bizMOB.TR.send({
    _sTRName: 'FMS3003',
    _oBody: {
      mtrNum: data.mtrNum,
      indicationAmt: data.indicationAmt,
      indicationAmtYn: data.indicationAmtYn,
      // mtrDetaLoc 없으면 서버에 안 감!
    },
  });
}
```

## 해결

`sendChangeResult` 내 `_oBody`에 `mtrDetaLoc`를 명시적으로 추가한다.

```javascript
bizMOB.TR.send({
  _sTRName: 'FMS3003',
  _oBody: {
    mtrNum: data.mtrNum,
    indicationAmt: data.indicationAmt,
    indicationAmtYn: data.indicationAmtYn,
    mtrDetaLoc: data.mtrDetaLoc,   // ← 이 줄 추가
  },
});
```

<Callout label="결론">
scgfms에서 새 필드를 서버로 보내려면 두 곳을 동시에 수정해야 한다: (1) `DEFAULT_PAYLOAD`에 초기값 추가, (2) TR 호출 함수의 `_oBody`에 필드 명시. 한 곳만 수정하면 화면에는 표시되지만 서버에는 전달되지 않거나, 반대로 서버에는 값이 있지만 화면에 없는 상태가 된다.
</Callout>

## 남은 의문

`page.payload` 전체를 그대로 TR에 넘기는 헬퍼가 있는지, 있다면 왜 이 TR에선 쓰지 않는지 확인 필요.
