## 관찰 / 문제

bizMOB + Swiper virtual slides 기반 카드 목록에서, form `<input>`이 아닌 팝업으로 특정 필드(예: `mtrDetaLoc`)를 인라인 편집하고 저장 버튼을 누르면 편집 값이 반영되지 않았다.

## 왜 그런가 / 원인

`page.payload`는 `refreshCardList(reset=true)` 또는 Swiper `slideChange` 이벤트 때만 `getPayload(data)`로 재구성된다.

```js
// slideChange — payload가 재구성되는 유일한 두 시점
page.payload = page.getPayload(page.filteredMeterList[curIdx]);
// or:
if (reset) page.payload = page.getPayload(page.filteredMeterList[initialIdx]);
```

팝업으로 값을 바꾼 뒤 `refreshCardList(false)`를 호출하면 Swiper가 재생성되지만 `reset=false`이므로 `page.payload`는 **갱신되지 않는다**. 저장 버튼은 `page.payload` 그대로를 IndexedDB에 쓰기 때문에 편집 값이 유실된다.

## 해결

팝업 confirm 핸들러 한 곳에서 세 군데를 모두 업데이트한다.

```js
$('#mtrDetaLocConfirm').bind('click', async function () {
  const newLoc = $('#mtrDetaLocInput').val();
  const mtrNum = page.filteredMeterList[page._editingMtrDetaLocIdx].mtrNum;

  // ① 메모리(filteredMeterList) — 카드 재렌더링용
  page.filteredMeterList = page.filteredMeterList.map((m) =>
    m.mtrNum === mtrNum ? { ...m, mtrDetaLoc: newLoc } : m
  );

  // ② 현재 payload — 저장 버튼이 이 객체를 그대로 사용함
  if (page.payload && page.payload.mtrNum === mtrNum) {
    page.payload.mtrDetaLoc = newLoc;
  }

  // ③ IndexedDB — 앱 재진입 후에도 유지
  const list = (await gasUtil.getLargeMeterData(page.db)) || [];
  await gasUtil.setLargeMeterData(
    list.map((m) => (m.mtrNum === mtrNum ? { ...m, mtrDetaLoc: newLoc } : m))
  );

  page.refreshCardList(false); // UI만 갱신
});
```

또한 `DEFAULT_PAYLOAD`에 해당 필드를 추가해 `getPayload` 호출 시 항상 포함되도록 보장한다.

```js
DEFAULT_PAYLOAD: {
  // ... 기존 필드
  mtrDetaLoc: '',
},
```

<Callout label="결론">
bizMOB Swiper 카드에서 form input 대신 팝업으로 데이터를 편집할 때는, confirm 핸들러 안에서 `filteredMeterList`, `page.payload`, IndexedDB 세 곳을 모두 즉시 업데이트해야 한다. 셋 중 하나라도 빠지면 저장 후 데이터가 유실된다.
</Callout>

## 남은 의문

`page.refreshCardList(false)` 후 Swiper가 재생성될 때 `slideChange`가 발화하지 않음을 이용한 것인데, 이 동작이 Swiper 버전에 따라 달라질 수 있는지는 확인하지 않았다.
